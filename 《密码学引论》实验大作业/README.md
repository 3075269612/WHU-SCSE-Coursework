# SM4 文件加密系统（C语言 + GTK 3 + OpenSSL）

## 项目简介

本项目是《密码学引论》课程大作业，完成以下四项要求：

| 要求 | 实现方式 |
|------|---------|
| 文件加密和解密 | SM4-CBC-CTS 加密/解密，RSA-2048 + OAEP 保护会话密钥 |
| 加解密速度统计 | `clock()` 计时，计算 MB/s 吞吐率，GUI 日志显示 |
| 密文链接 + 密文挪用 | **密文链接**：CBC 模式；**密文挪用**：CBC-CS1（≥16 字节零膨胀） |
| 人机界面 | GTK 3 图形界面，文件选择器、按钮操作、滚动日志 |

## 复现步骤

### 1. 安装依赖

```bash
sudo apt update
sudo apt install -y build-essential libgtk-3-dev libssl-dev pkg-config
```

### 2. 编译

```bash
cd "《密码学引论》实验大作业"
make
```

编译成功后将生成 `sm4_gui` 可执行文件。

### 3. 运行

```bash
./sm4_gui
```

### 4. 操作流程

1. 点击 **"生成RSA密钥对"** — 生成 `public.pem` 和 `private.pem`
2. 点击 **"选择文件"** — 选择要加密的文件
3. 点击 **"加密文件"** — 生成 `原文件名.enc` 密文文件
4. 选择 `.enc` 文件，点击 **"解密文件"** — 生成 `原文件名.enc.dec` 解密文件
5. 使用 `diff 原文件 原文件名.enc.dec` 验证解密正确性

### 5. 命令行快速验证

```bash
# 创建测试文件
echo "Hello SM4 CBC-CTS!" > test.txt

# 编译运行
make && ./sm4_gui

# 验证解密结果
diff test.txt test.txt.enc.dec && echo "加解密完全正确"
```

## 使用说明

- **首次使用**需要先生成 RSA 密钥对（点击"生成RSA密钥对"按钮）
- **密钥对只需生成一次**，`public.pem` 用于加密，`private.pem` 用于解密
- **私钥 `private.pem` 必须妥善保管**，解密时需要用到
- 加密后的 `.enc` 文件可在不同机器间传输，只要持有 `private.pem` 即可解密
- 日志区显示明文/密文大小对比，可直观验证 CTS 零膨胀特性

## 技术实现

| 技术 | 详情 |
|------|------|
| 分组密码 | **SM4**（128 bit 密钥，128 bit 分组，32 轮非平衡 Feistel） |
| 工作模式 | **CBC**（密文分组链接，密文链接技术） |
| 短块处理 | **CBC-CS1**（NIST SP 800-38A Addendum，密文挪用技术） |
| 密钥保护 | **RSA-2048 + OAEP 填充**（混合加密体制） |
| 随机数 | OpenSSL `RAND_bytes()`（CSPRNG） |
| 图形界面 | **GTK 3** |
| 密码库 | OpenSSL EVP API |

### CTS 零膨胀

- **明文 ≥ 16 字节**：密文长度 = 明文长度（零膨胀）
- **明文 < 16 字节**：回退 PKCS#7 填充（密文 = 16 字节，1~15 字节膨胀）

> CTS 的完整算法推导和安全性分析见 `SM4实验报告.md` 第三章。

## 文件格式

加密后 `.enc` 文件的二进制结构：

```
偏移    长度    字段
0       8       MAGIC 标识 "SM4ENC02"
8       4       加密密钥长度（大端 uint32）
12      4       IV 长度（大端 uint32，固定 16）
16      var     RSA-OAEP 加密的 SM4 会话密钥（RSA-2048 → 256 字节）
16+k    16      随机 IV（初始化向量）
32+k    1       模式标识（0x01=CTS, 0x00=PKCS#7）
33+k    var     SM4-CBC-CTS 密文（长度 = 明文长度）
```
